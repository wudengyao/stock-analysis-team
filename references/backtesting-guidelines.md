## 回测目标

### 主要目标
1. **验证分析准确率**：评估历史分析的预测准确性
2. **计算方向胜率**：统计预测方向与实际走势的吻合度
3. **测量止盈止损命中率**：评估交易策略的执行效果
4. **识别策略弱点**：发现分析框架的不足之处

### 回测范围
- 时间范围：建议回测最近1-3年
- 样本数量：至少20个分析案例
- 市场周期：覆盖牛市、熊市、震荡市

## 核心指标

### 1. 方向胜率（Directional Accuracy）
**定义**：预测方向（看涨/看跌）与实际方向一致的比率

**计算公式**：
```
方向胜率 = 预测正确的次数 / 总预测次数 × 100%
```

**判断标准**：
- >70%：优秀
- 60%-70%：良好
- 50%-60%：一般
- <50%：需要改进

**示例**：
```
历史分析记录：
- 2023-01-15：看涨 → 实际上涨 ✓
- 2023-02-10：看跌 → 实际下跌 ✓
- 2023-03-05：看涨 → 实际下跌 ✗
- 2023-04-01：看涨 → 实际上涨 ✓
- 2023-05-08：看跌 → 实际下跌 ✓

方向胜率 = 4/5 × 100% = 80%
```

### 2. 止盈命中率（Take-Profit Hit Rate）
**定义**：预测止盈位被触及的概率

**计算公式**：
```
止盈命中率 = 触及止盈位的次数 / 设置止盈位的次数 × 100%
```

**判断标准**：
- >60%：优秀
- 40%-60%：良好
- 20%-40%：一般
- <20%：需要改进

**示例**：
```
历史分析记录：
- 2023-01-15：止盈位150元 → 最高触及152元 ✓
- 2023-02-10：止盈位200元 → 最高触及198元 ✗
- 2023-03-05：止盈位180元 → 最高触及185元 ✓
- 2023-04-01：止盈位220元 → 最高触及218元 ✗

止盈命中率 = 2/4 × 100% = 50%
```

### 3. 止损命中率（Stop-Loss Hit Rate）
**定义**：预测止损位被触及的概率

**计算公式**：
```
止损命中率 = 触及止损位的次数 / 设置止损位的次数 × 100%
```

**判断标准**：
- <20%：优秀（说明预测准确，很少触发止损）
- 20%-40%：良好
- 40%-60%：一般
- >60%：需要改进（止损设置过于激进）

**示例**：
```
历史分析记录：
- 2023-01-15：止损位120元 → 最低触及118元 ✓
- 2023-02-10：止损位150元 → 最低触及155元 ✗
- 2023-03-05：止损位140元 → 最低触及138元 ✓
- 2023-04-01：止损位190元 → 最低触及195元 ✗

止损命中率 = 2/4 × 100% = 50%
```

### 4. 平均盈亏比（Average Risk-Reward Ratio）
**定义**：平均盈利与平均亏损的比值

**计算公式**：
```
平均盈亏比 = 平均盈利金额 / 平均亏损金额
```

**判断标准**：
- >2：优秀
- 1.5-2：良好
- 1-1.5：一般
- <1：需要改进

### 5. 最大回撤（Maximum Drawdown）
**定义**：从峰值到谷底的最大跌幅

**计算公式**：
```
最大回撤 = (峰值 - 谷底) / 峰值 × 100%
```

**判断标准**：
- <10%：优秀
- 10%-20%：良好
- 20%-30%：一般
- >30%：需要改进

### 6. 夏普比率（Sharpe Ratio）
**定义**：单位风险的超额收益

**计算公式**：
```
夏普比率 = (平均收益率 - 无风险收益率) / 收益率标准差
```

**判断标准**：
- >2：优秀
- 1-2：良好
- 0.5-1：一般
- <0.5：需要改进

## 回测方法

### 方法1：历史分析追踪法
**适用场景**：已有历史分析记录的情况

**步骤**：
1. 收集历史分析记录（日期、股票代码、预测方向、止盈止损位）
2. 获取对应的历史行情数据
3. 逐一验证每个分析的准确性
4. 统计各项指标

**示例**：
```
# 历史分析记录格式
{
  "date": "2023-01-15",
  "symbol": "600519.SH",
  "prediction": "看涨",
  "target_price": 1800,
  "stop_loss": 1500,
  "actual_high": 1820,
  "actual_low": 1480,
  "outcome": "盈利"
}
```

### 方法2：模拟回测法
**适用场景**：需要验证分析框架的长期效果

**步骤**：
1. 设定回测参数（时间范围、初始资金、仓位管理）
2. 应用分析框架对历史数据进行分析
3. 模拟执行交易决策
4. 计算回测结果

**参数设定**：
```python
# 回测参数示例
backtest_params = {
    "start_date": "2021-01-01",
    "end_date": "2023-12-31",
    "initial_capital": 1000000,  # 100万初始资金
    "max_position": 0.3,  # 单只股票最大仓位30%
    "stop_loss_pct": 0.08,  # 止损8%
    "take_profit_pct": 0.20  # 止盈20%
}
```

### 方法3：蒙特卡洛模拟法
**适用场景**：评估策略的稳健性和极端情况表现

**步骤**：
1. 随机生成多个价格路径
2. 应用交易策略
3. 统计结果的分布情况
4. 评估策略的可靠性

## 验证流程

### 流程图
```
1. 收集历史分析数据
   ↓
2. 获取对应历史行情
   ↓
3. 逐一验证分析准确性
   ↓
4. 统计核心指标
   ↓
5. 生成回测报告
   ↓
6. 识别策略弱点
   ↓
7. 提出改进建议
```

### 详细步骤

**步骤1：数据收集**
- 收集至少20个历史分析案例
- 每个案例包含：日期、股票代码、预测方向、止盈止损位、分析理由
- 数据来源：SKILL执行日志、研究报告存档

**步骤2：行情获取**
- 使用 `scripts/market_data_fetcher.py` 获取历史数据
- 时间范围：分析日期后至少30个交易日
- 需要获取：开盘价、最高价、最低价、收盘价

**步骤3：准确性验证**
- 方向验证：预测方向与实际走势是否一致
- 止盈验证：最高价是否触及止盈位
- 止损验证：最低价是否触及止损位
- 盈亏计算：如果持有至止盈/止损，盈亏是多少

**步骤4：指标统计**
- 计算方向胜率
- 计算止盈/止损命中率
- 计算平均盈亏比
- 计算最大回撤

**步骤5：报告生成**
- 汇总所有指标
- 分析趋势和规律
- 识别策略弱点

**步骤6：改进建议**
- 针对低胜率方向提出改进
- 针对低止盈命中率优化止盈位设置
- 针对高止损命中率调整止损策略

## 注意事项

### 回测陷阱
1. **过拟合**：过度优化策略，导致回测结果优异但实盘表现差
2. **生存偏差**：只回测存活至今的股票，忽略了退市股票
3. **未来函数**：使用了未来数据，导致回测结果虚高
4. **交易成本忽略**：未考虑手续费、滑点等交易成本

### 改进方法
1. **样本外测试**：保留部分数据不参与策略优化，用于验证
2. **参数敏感性测试**：测试策略对参数变化的敏感度
3. **跨周期验证**：在不同市场周期下验证策略有效性
4. **考虑交易成本**：将手续费、滑点纳入回测计算

### 持续监控
- 每月更新回测报告
- 对比实盘与回测结果的差异
- 及时调整策略参数
- 识别新的市场环境变化
